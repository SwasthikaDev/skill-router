How it works

Three steps, fully autonomous — no human in the loop.

1

Describe the need

Send POST /find with a plain-language need. No auth, no setup.

2

Get a ranked match

Deterministic scoring over the live registry returns the best skills, each with why it matched.

3

Run the call plan

Each result includes a paste-ready curl and a link to that skill’s SKILL.md.

Why we built it

The NANDA registry already lists dozens of agent services, but nothing lets an agent find the right one while it is running — it has to already know the service exists and how to call it. That missing discovery step is what keeps agents from using each other's capabilities. Skill-Router fills it: describe a need, get the service and a ready-to-run call. Almost every agent task starts with “which service do I call?”, so this is the piece the rest builds on.

Built to scale

Stateless, deterministic, and cache-friendly — the same properties that let DNS resolvers serve the whole internet.

Stateless fleet

No per-request state. Any node answers any /find, so it sits behind a load balancer and autoscales on request volume — add or drop nodes freely.

Cheap per query

Ranking is a deterministic IDF pass over an in-memory index of lean records, so a lookup is millisecond-scale. A large registry shards by tag or namespace.

Read-scale at the edge

/find is idempotent and deterministic, so identical needs cache at a CDN or edge and never hit origin — the read-scaling DNS gets from TTL caching.

Graceful degradation

A bundled snapshot backs the live registry, so an upstream outage degrades to slightly stale results instead of downtime. A keep-warm ping avoids cold starts.

Deterministic responses · self-correcting error messages · health check · no API key, no rate limits.
